## 1. 앱 개요

**Snap URL**은 AI 기반 URL 수집/정리/요약 서비스의 iOS 클라이언트이다. 사용자가 URL을 입력하면 백엔드가 비동기로 콘텐츠를 수집하고, AI 요약을 생성하여 저장한다.

### 핵심 기능

- URL 입력 및 비동기 수집 요청
- 비동기 작업(Job) 상태 추적 (폴링)
- 수집된 문서 목록 조회 (카테고리 필터링)
- 문서 상세 보기 (요약, 관련 링크)
- 문서 수정 (제목, 설명, 관련 링크 편집)
- 문서 삭제
- 실패 시 재시도

> 백엔드 API 상세 명세는 [backend-async-ingest-spec-v1.md](./backend-async-ingest-spec-v1.md) 참조.

## 2. 화면 구성

### 2.1 메인 화면 — URL 입력

**목적**: URL을 입력하고 수집을 시작하는 진입 화면.

**UI 구성**:

- **Status Bar**: 시간(9:41), 시그널/배터리 아이콘
- **타이틀**: "Snap URL" (Inter, 40pt, ExtraBold)
- **URL 입력 필드**:
  - 배경: `$input-background` (#FFFFFF)
  - cornerRadius: 16
  - 높이: 157px
  - 패딩: [16, 24]
  - Placeholder: "https://example.com" (Inter, 17pt, Regular)
  - 키보드 타입: URL, 자동 대문자화 비활성화
- **수집 시작 버튼**:
  - 배경: `$primary-color`
  - 텍스트: "수집 시작" (Inter, 17pt, SemiBold, 흰색)
  - cornerRadius: 16
  - 높이: 56px
  - 너비: fill
  - URL 유효성 검사 통과 시 활성화
- **Spacer**: 버튼과 탭 바 사이 여백
- **탭 바**: "홈" 탭 활성 상태 (하단 pill-style, 섹션 2.5 참조)

**상태 처리**:

- URL 제출 시 → `POST /ingest` 호출 → Job ID 수신 → 처리 중 화면으로 전환
- 버튼 로딩 상태: 스피너 또는 비활성화

### 2.2 처리 중 화면 — Job 상태

**목적**: URL 처리 진행 상황을 실시간으로 표시.

> 이 화면은 디자인 파일에 아직 미포함. 아래는 기능 명세 기준이며 추후 디자인 확정 필요.

**UI 구성**:

- 상단: 뒤로가기 버튼
- 중앙:
  - URL 표시 (`normalized_url`)
  - 진행 상태 인디케이터
    - `queued`: "대기 중..." + 스피너
    - `running`: "처리 중..." + 진행 바 (애니메이션)
    - `succeeded`: "완료!" + 체크마크 → 자동으로 문서 상세로 이동
    - `failed`: "실패" + 에러 메시지 + 재시도 버튼

**폴링 전략**:

| 폴링 횟수 | 간격 |
|-----------|------|
| 1~5회 | 1초 |
| 6~15회 | 2초 |
| 이후 | 3초 (최대 60초까지) |

### 5.4 접근성

- VoiceOver 지원
- Dynamic Type 지원
- 색상 대비 준수 (WCAG AA)

### 5.5 삭제 확인

## 6. 클라이언트 API 연동 플로우

### 6.1 URL 제출 → 문서 생성

```
1. 사용자 URL 입력
2. 클라이언트 유효성 검사
3. POST /ingest 호출
4. 202 Accepted + Job 리소스 수신
5. 처리 중 화면 표시
6. GET /ingest-jobs/:id 폴링 시작 (섹션 2.2 폴링 전략)
7. status === "succeeded" 시
   → job.document_id 로 GET /documents/:id 호출
   → 문서 상세 화면 표시
8. status === "failed" 시
   → error.code 기준 메시지 표시
   → error.retryable === true 이면 재시도 버튼 제공
```

### 6.2 문서 목록 조회

```
1. 문서 탭 진입 또는 앱 시작
2. GET /documents?limit=20 호출
3. 카드 리스트 렌더링
4. 카테고리 태그 선택 시 → 로컬 필터링 또는 재요청
5. Pull-to-refresh → 전체 리로드
6. 하단 스크롤 → 추가 페이지 로드
```

### 6.3 문서 수정

```
1. 문서 상세 화면에서 "수정" 버튼 탭
2. 문서 수정 화면으로 push
3. 사용자가 제목/설명/관련 링크 수정
4. "저장" 버튼 탭
5. PATCH /documents/:id 호출
   - Request Body: { title, description, links }
6. 성공 시 → 문서 상세 화면으로 pop (업데이트된 데이터)
7. 실패 시 → 에러 메시지 표시
```

### 6.4 문서 삭제

```
1. 문서 상세 화면에서 "삭제" 버튼 탭
2. 확인 다이얼로그 표시
3. 사용자가 "삭제" 확인
4. DELETE /documents/:id 호출
5. 성공 시 → 문서 목록 화면으로 pop
6. 실패 시 → 에러 메시지 표시
```
